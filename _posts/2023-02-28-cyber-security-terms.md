# Terms
**SOAR** - Security Orchestration, Automation and Response: A platform that helps automate incident response processes by integrating various security tools, streamlining workflows, and automating repetitive tasks.

**SIEM** - Security Information and Event Management: A system that collects and aggregates security data from various sources, such as network traffic, system logs, and security devices. It then uses analytics to detect and respond to security threats.

**IOC** - Indicators of Compromise: Pieces of information that indicate a security threat, such as a file hash, IP address, or domain name associated with malware.

**EDR**- Endpoint Detection and Response: A security solution that monitors endpoint devices, such as laptops and servers, for signs of malicious activity.

**DLP** - Data Loss Prevention: A set of tools and techniques used to prevent sensitive data from being leaked or stolen, either intentionally or unintentionally.

**VPN** - Virtual Private Network: A network technology that allows users to securely access resources over the internet by creating an encrypted tunnel between the user's device and the resource they are accessing.

**MFA** - Multi-Factor Authentication: A security mechanism that requires users to provide two or more authentication factors, such as a password and a one-time code sent to their phone, to gain access to a system or resource.

**IDS** - Intrusion Detection System: A system that monitors network traffic for signs of unauthorized access or malicious activity.

**IPS** - Intrusion Prevention System: A system that detects and prevents malicious activity on a network by actively blocking traffic that matches known threat signatures or behaviors.

**TIP** - Threat Intelligence Platform: A software tool that helps organizations collect, analyze, and share threat intelligence, including IOCs, to stay up-to-date on the latest security threats.

**Ransomware** - Malicious software that encrypts files on a victim's computer and demands payment in exchange for the decryption key.

**Phishing** - A type of social engineering attack where an attacker sends an email or other message that appears to be from a legitimate source, but is actually designed to trick the recipient into revealing sensitive information.

**Malware** - Short for "malicious software, this term refers to any software designed to cause harm to a computer system or network.

**Botnet** - A network of infected computers that can be remotely controlled by an attacker to carry out various malicious activities, such as launching DDoS attacks or stealing sensitive data.

**Two-Factor Authentication (2FA)** - A security mechanism that requires users to provide two forms of authentication to access a system or resource, such as a password and a one-time code sent to their phone.

**Advanced Persistent Threat (APT)** - A sophisticated and persistent cyber attack carried out by a skilled and motivated adversary, often with the goal of stealing sensitive data or disrupting operations.

**Zero-Day Vulnerability** - A software vulnerability that is unknown to the vendor or developers, and can be exploited by attackers before a patch or fix is available.

**Patch Management** - The process of applying updates, patches, and fixes to software and systems to address known vulnerabilities and improve security.

**Social Engineering** - The use of psychological manipulation to trick individuals into divulging sensitive information or performing actions that may not be in their best interest.

**Security Operations Center (SOC)** - A team or department responsible for monitoring and responding to security threats, typically using tools such as SIEM and SOAR platforms.

**Denial of Service (DoS)** - An attack that aims to disrupt or deny access to a system or network, typically by overwhelming it with traffic or requests.

**Distributed Denial of Service (DDoS)** - A type of DoS attack that is carried out using multiple compromised systems or devices, often coordinated using a botnet.

**Encryption** - The process of converting data into a coded form that can only be read or decrypted with a key or password.

**Decryption** - The process of converting encrypted data back into its original, readable form.

**Network Segmentation** - The practice of dividing a network into smaller, isolated segments to improve security and limit the impact of any potential breach.

**Patch** - A software update that fixes a security vulnerability or other issue.

**Exploit** - A piece of software or code that takes advantage of a vulnerability or weakness in a system to carry out a specific action, such as gaining unauthorized access.

**Vulnerability** - A weakness or flaw in a system or software that can be exploited by attackers to gain access or carry out malicious activities.

**Incident Response** - The process of responding to and mitigating the impact of a cybersecurity incident, such as a breach or attack.

**Threat Intelligence** - Information about known or potential cybersecurity threats, including tactics, techniques, and procedures used by attackers.

**Zero Trust** - A security model that assumes all devices, users, and traffic are untrusted by default, and requires strict authentication and access controls for all resources.

**Defense in Depth** - A security strategy that involves layering multiple security controls and mechanisms to protect against a range of threats.

**Red Team** - A group of skilled cybersecurity professionals tasked with simulating an attack or breach to identify weaknesses in an organization's defenses.

**Privileged access management (PAM)** -  is an identity security solution that helps protect organizations against cyberthreats by monitoring, detecting, and preventing unauthorized privileged access to critical resources. PAM works through a combination of people, processes, and technology and gives you visibility into who is using privileged accounts and what they are doing while they are logged in. Limiting the number of users who have access to administrative functions increases system security while additional layers of protection mitigate data breaches by threat actors.

**Blue Team** - A group of cybersecurity professionals responsible for defending an organization's systems and networks, often using security tools such as SIEM and SOAR platforms.

**Incident Command System (ICS)** - A standardized framework for managing emergency situations and coordinating response efforts among multiple organizations and agencies.

**Security Assessment** - A comprehensive evaluation of an organization's security posture, typically including vulnerability assessments, penetration testing, and other security tests.

**Security Operations** - The day-to-day activities and processes involved in maintaining and monitoring an organization's security posture, often involving security tools such as firewalls, antivirus software, and intrusion detection systems.

**Risk Management** - The process of identifying, assessing, and prioritizing risks to an organization's assets and implementing strategies to mitigate those risks.

**Security Awareness Training** - Education and training programs designed to help employees and other stakeholders understand and follow best practices for information security.

**Security Information Management (SIM)** - The process of collecting, analyzing, and reporting on security-related data to support decision-making and improve security posture.
